# ניהול הגעת מטופלים — Patient Attendance Manager

תוכנת דסקטופ לניהול הגעת מטופלים לטיפולים, בנויה ב-Python עם PySide6 (Qt).

## מבנה הפרויקט

```
clinic_app/
├── app.py          # נקודת הכניסה להרצת התוכנה
├── ui.py           # כל רכיבי הממשק הגרפי (PySide6)
├── models.py       # לוגיקה עסקית וייצוג הנתונים (ללא תלות ב-GUI)
├── storage.py       # שמירה/טעינה של נתונים לקובץ JSON
├── settings.py      # קבועים, נתיבי קבצים, עיצוב (QSS)
├── requirements.txt
└── README.md
```

הפרדה ברורה בין שכבות: **UI** (`ui.py`) ↔ **לוגיקה** (`models.py`) ↔ **שמירה** (`storage.py`).
זה מאפשר להוסיף בעתיד יכולות נוספות (למשל: ייצוא ל-Excel, דוחות, גיבוי בענן וכו') מבלי לשבש קוד קיים.

## התקנה והרצה (סביבת פיתוח)

```bash
pip install -r requirements.txt
python app.py
```

בהרצה הראשונה תיווצר אוטומטית ריק, ולאחר מכן ייווצר קובץ `clinic_data.json`
לצד הקוד (או לצד קובץ ה-EXE לאחר אריזה), ובו יישמרו כל הנתונים אוטומטית
אחרי כל שינוי (הוספת מטופל, סימון הגעה, שינוי מחיר וכו').

## שימוש בתוכנה

1. **טבלת המפגשים** (מרכז המסך) — שמות המטופלים מוצגים בעמודה הקבועה מימין
   (Header של השורות), התאריכים בשורה העליונה הקבועה. לחיצה על תא הופכת
   אותו ל-✔ ולהיפך.
2. **ניהול מטופלים** (פאנל צד) — הוספה/עריכה/מחיקה של מטופלים.
3. **ניהול תאריכים** (פאנל צד) — הוספה/עריכה/מחיקה של תאריכים (בפורמט
   `DD/MM/YYYY`, למשל `01/07/2026`). התאריכים ממוינים אוטומטית כרונולוגית.
4. **מחיר וסיכום** — הזנת מחיר לטיפול; סה"כ מפגשים והכנסה מתעדכנים מיידית,
   הן בפאנל הצד והן בשורת הסיכום התחתונה.

## אריזה לקובץ EXE יחיד עם PyInstaller

### התקנת PyInstaller

```bash
pip install pyinstaller
```

### פקודת האריזה המדויקת (Windows, קובץ EXE יחיד, ללא חלון קונסולה)

הרץ מתוך תיקיית הפרויקט (`clinic_app/`):

```bash
pyinstaller --noconfirm --onefile --windowed --name "ClinicAttendance" --icon "icon.ico" app.py
```

הסבר הדגלים:

| דגל | משמעות |
|---|---|
| `--onefile` | אורז הכל (כולל Python ו-Qt) לקובץ EXE יחיד |
| `--windowed` | ללא חלון קונסולה שחור ברקע (מתאים לאפליקציית GUI) |
| `--name` | שם קובץ ה-EXE הסופי |
| `--icon` | אייקון מותאם אישית (ראה הערה למטה) |

לאחר ריצה מוצלחת, קובץ ה-EXE יימצא בתיקייה:

```
dist/ClinicAttendance.exe
```

### הכנת אייקון מותאם אישית

- הכן קובץ אייקון בפורמט `.ico` (בגודל מומלץ הכולל כמה רזולוציות: 16x16 עד
  256x256). ניתן להמיר תמונת PNG/JPG לקובץ ICO באמצעות כלים חינמיים כמו
  [icoconvert.com](https://icoconvert.com) או תוכנת עריכת תמונה.
- שמור את הקובץ בשם `icon.ico` בתוך תיקיית הפרויקט, ואז הרץ את פקודת
  ה-PyInstaller שלמעלה כפי שהיא (היא כבר מפנה אליו).
- אם אין ברשותך אייקון מותאם, ניתן פשוט להשמיט את `--icon "icon.ico"`
  מהפקודה, וייעשה שימוש באייקון ברירת המחדל.

### המלצות חשובות לאריזה נקייה

1. **הרץ מתוך סביבה וירטואלית נקייה** (`venv`) שמכילה רק את החבילות
   הדרושות (`pip install -r requirements.txt pyinstaller`), כדי לצמצם את
   גודל קובץ ה-EXE ולמנוע קונפליקטים של גרסאות.
2. **בדוק את קובץ ה-EXE במחשב "נקי"** (ללא Python מותקן) כדי לוודא שהוא
   רץ עצמאית.
3. **מיקום קובץ הנתונים** — הקוד ב-`settings.py` דואג לכך שקובץ
   `clinic_data.json` יישמר תמיד לצד קובץ ה-EXE (ולא בתיקיית זמנית פנימית
   של PyInstaller), כך שהנתונים לא "יאבדו" בין הרצות.
4. אם ברצונך גרסה עם תיקיית קבצים נלווית במקום קובץ יחיד (טעינה מהירה
   יותר בהרצה חוזרת), השתמש ב-`--onedir` במקום `--onefile`.
5. לאחר שינויים בקוד, מומלץ למחוק את התיקיות `build/` ו-`dist/` וכן את
   קובץ ה-`.spec` שנוצר, ולהריץ את הפקודה מחדש, כדי להימנע מקבצים ישנים
   שנשארו בזיכרון המטמון של PyInstaller:

```bash
rm -rf build dist ClinicAttendance.spec   # macOS/Linux
# או
rmdir /s /q build dist & del ClinicAttendance.spec   # Windows
```

## הרחבות עתידיות אפשריות

הודות למבנה המודולרי, קל להוסיף בעתיד:

- ייצוא הטבלה לקובץ Excel (`openpyxl`) או PDF.
- מסך התחברות/הרשאות למספר משתמשים.
- גיבוי אוטומטי לענן (Google Drive / Dropbox API).
- חיפוש וסינון מטופלים בטבלה.
- תמיכה במספר "מטפלים" עם טבלאות נפרדות.

כדי להוסיף יכולת חדשה: הוסף פונקציונליות עסקית ל-`models.py`, חשוף אותה
דרך ווידג'ט חדש או קיים ב-`ui.py`, וחבר בין השניים ב-`MainWindow`
(`_connect_signals`) — מבלי לגעת בשאר הקוד.
